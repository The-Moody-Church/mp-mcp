# 2026-04-17 — Session Fix & Domain Tools

## Summary

Found and fixed the root cause of all connection failures (stale MCP sessions), confirmed that server instructions are NOT size-limited, and shipped domain-specific tools for church staff use.

## Root Cause: Session Management

Added MCP response body logging which revealed the actual errors Claude Desktop was seeing:
- `"Server already initialized"` (400) — transport rejected re-initialization
- `"Mcp-Session-Id header is required"` (400) — requests without session header rejected

After any redeploy, token refresh, or disconnect/reconnect, Claude Desktop would send requests that didn't match the server's in-memory sessions. The server returned 400 errors which Claude Desktop interpreted as "Authorization failed."

**Fix:** Always create a fresh transport when no matching session exists. Index transports by both user ID and session ID. Any request — initialize or not — gets a working transport instead of a 400 error.

## Instructions Size Myth Debunked

Earlier testing suggested large server `instructions` broke Claude Desktop. After fixing the session management, we confirmed 8k instructions work fine with a fresh connector add. The failures were always the session issue, not the instructions size. The official docs confirm no documented size limits.

## Domain Tools (PR #12)

Added 6 domain-specific tools that encode MP schema knowledge:

| Tool | Purpose | Key FK Joins Baked In |
|------|---------|----------------------|
| `find_people` | Search by name/email/phone | Searches Nickname, First_Name, Last_Name individually |
| `get_person_details` | Full profile + groups + events | Contacts → Household → Address, Member_Status, Gender, Marital_Status |
| `search_groups` | Find groups by name/type/ministry | Group_Type, Ministry, Congregation, Meeting_Day, Meeting_Frequency |
| `get_group_roster` | Members with roles | Contact display name, Role_Title, Group_Role_Type |
| `search_events` | Find events by date/name/program | Event_Type, Program, Congregation |
| `get_event_attendance` | Individual + aggregate attendance | Participation_Status, Event_Metrics for headcount |

Generic tools (query_table, get_record, list_tables, describe_table) kept as power-user escape hatches.

## FK Join Bugs Fixed

From actual error logs:
- `Contacts.Contact_ID` — must prefix to avoid ambiguity when joining
- `Congregation_ID` is on Households, not Contacts — use `Household_ID_Table_Congregation_ID_Table.Congregation_Name`
- `Participation_Status_ID` — must prefix with `Event_Participants.` to disambiguate
- Nested FK joins in `$select` don't work (e.g., `Event_ID_Table.Event_Type_ID_Table.Event_Type`). Only underscore-chained joins work.

## Architecture Change

```
src/
├── server.ts              # McpServer setup, instructions, tool registration
├── tools/
│   ├── auth.ts            # Shared getAuthFromExtra()
│   ├── people.ts          # find_people, get_person_details
│   ├── groups.ts          # search_groups, get_group_roster
│   ├── events.ts          # search_events, get_event_attendance
│   └── generic.ts         # list_tables, describe_table, query_table, get_record
├── transport.ts           # MP API requests (unchanged)
├── config.ts              # Allowlist + env config (unchanged)
└── index.ts               # Express server, auth, session management
```

## Decisions

- **No giving/donation tools** — excluded by design, not just by allowlist
- **Connector reconnect requires personal settings remove/re-add** for new tools — org connector stays, but personal tool cache needs clearing
- **Response body logging kept** for now — useful for debugging FK join issues
