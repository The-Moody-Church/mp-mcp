# 2026-04-14 — Instructions & Stability

## Summary

Attempted to add data presentation instructions (FK join guidance, donation policy, table schema). Discovered that Claude Desktop has strict size limits on MCP server responses — large `instructions` and tool `description` fields break connection/tool discovery. Reverted to the working baseline. Tools are working end-to-end.

## What We Learned

### Claude Desktop Size Limits
- **Server `instructions`**: 8k chars broke reconnect entirely. 710 chars also broke it. The field itself may be problematic.
- **Tool `description`**: 1758 chars on query_table prevented tool discovery (tools "connected" but invisible). Original ~300 char descriptions work.
- **Stateless transport** (`sessionIdGenerator: undefined`): Caused tools not to surface — Claude Desktop needs the SSE stream (GET /mcp) from stateful mode.

### Error Visibility
- Added `[MP API]` error logging to transport.ts (PR #6) — tool errors that were previously silent are now visible in container logs.
- Added `[Tool]` logging to list_tables handler for debugging.
- Example error caught: Claude tried `Group_Type_ID_Table` FK join on Group_Participants (wrong table — that FK is on Groups, not Group_Participants). Also hit ambiguous column names (Start_Date exists on both Group_Participants and joined Groups).

### Reconnect Issues
- After any container restart, Claude Desktop's cached MCP session ID becomes invalid → 404 → "auth failed" error.
- Workaround: remove connector and re-add it. Not sustainable for production.
- Stateless transport was attempted as a fix but broke tool discovery.
- This remains an open issue for the reconnect-after-deploy scenario.
