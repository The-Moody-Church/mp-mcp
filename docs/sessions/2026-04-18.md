## Pass 1 ‚Äî PR #13 (`claude/security-audit-sec-fixes`)

Targeted fixes requested by the user.

- **HIGH** ‚Äî `/register` was echoing `OIDC_CLIENT_SECRET` to unauthenticated registrants and accepting arbitrary `redirect_uris`. Registrants now get a fresh random secret; redirect URIs must be `https://` and on an allowlist (`https://claude.ai/api/mcp/auth_callback` + `ALLOWED_REDIRECT_URIS`); client store capped at 1000 with oldest-eviction.
- **CRITICAL** ‚Äî OAuth proxy logged full request bodies (auth codes, client secrets, refresh tokens) and truncated token-response fields. `verifyAccessToken` logged a token prefix. All removed.
- **MEDIUM** ‚Äî Request middleware buffered and logged MCP response bodies (PII from people/group queries). Reduced to method/path.
- **HIGH** ‚Äî `ALLOWED_USER_GROUP_IDS` group check fell through on any failure (non-OK response, zero-row lookup, network error). Now fail-closed.
- **MEDIUM** ‚Äî Date inputs now require `YYYY-MM-DD`. New `escapeLikeValue()` helper escapes `%`, `_`, `[` in addition to doubling quotes; routed every LIKE call site through it.
- **LOW** ‚Äî Deleted unused `src/auth/oidc.ts` and `src/auth/session.ts` (never imported); removed `SESSION_SECRET` from config/env.example/README; dropped `version` from `/health`.

## Pass 2 ‚Äî `claude/security-audit-pass-2`

Comprehensive audit across SQL, MCP, API, OAuth, PII, and dependencies. Seventeen findings; the six above plus eleven new ones.

### HIGH

- **#1 `describe_table` leaking PII** ‚Äî was calling the target table with `$top: 1` and returning the actual row values as "sampleValue". Now returns only field name + inferred type.
- **#2 Prune default allowlist** ‚Äî removed `dp_Users`, `Background_Checks`, `Form_Responses` from `table-access.example.json`. `dp_Users` was the big one (password-hash columns, reset tokens); the group-check path queries it directly outside the allowlist, so auth is unaffected. README documents what was removed and why.
- **#3 `query_table` hardening** ‚Äî accepted option (c): document the trust model. `query_table`'s raw `$filter`/`$select`/`$orderby` pass-through is the ad-hoc escape hatch. MP's role-based security is the authoritative column/join gate once a table is allowlisted. Written up in `docs/security-posture.md`.

### MEDIUM

- **#4/#5 PII in logs** ‚Äî MP error bodies echo user-supplied filter/select fragments (names, emails). Truncated to 200 chars in `transport.ts` before logging or re-raising. Per-request user log switched from `userName` to `userId` (opaque GUID).
- **#7 Transports map leak** ‚Äî `handleMcp` only cleaned up on `transport.onclose`; abandoned connections stayed forever. Added per-user activity timestamps, 30-min idle sweeper (unref'd), 500-user cap with oldest-eviction.
- **#8/#9 Rate limit and userinfo cache** ‚Äî SDK only rate-limits `/register` and `/token`. Added 120 req/min per-token limit on `/mcp` and `/`, keyed by `sha256(token)`. `verifyAccessToken` now caches `AuthInfo` for 60s per token hash (cap 10k, oldest-eviction, JWT exp honored).

### LOW ‚Äî batched cleanup

- **#10** Dockerfile now runs as `node:node` (not root).
- **#11** Dead `originalClientStore` assignment removed.
- **#13** `get_record`'s `id` now `z.number().int().positive()`.
- **#14** `npm audit fix` cleared GHSA-458j-xx4x-4375 (transitive hono).
- **#15** Removed unused `openid-client` dep.
- **#17** Comment on `trust proxy = 1` explaining the single-hop (Cloudflare tunnel) assumption.

## Accepted risks

Documented in `docs/security-posture.md`:

- **#6 Prompt injection via MP freeform data** ‚Äî structural mitigation already in place (`Form_Responses` excluded by default, donations out entirely). Claude's own instruction-following is the primary defense; a code-level strip/filter would be partial and fragile. Tripwire: a demonstrated exploit or pressure to re-add freeform tables by default.
- **#12 `getClient` returns configured `OIDC_CLIENT_SECRET`** ‚Äî not exposed over any current HTTP path (SDK's `/register` only returns secrets to registrants; no RFC 7592 endpoint). Tripwire: adding any client-read endpoint or SDK upgrade that exposes this.
- **#16 Silent fallback to example `table-access.json` in container** ‚Äî the example is itself conservative; correct place to enforce presence is deployment config, not app code. Tripwire: operator reporting an unexpected exposure.
- **N2 60-second revocation window** (added pass 3) ‚Äî verifyCache caches the allowlist check for up to 60s, so removing a user from `ALLOWED_USER_GROUP_IDS` without also disabling their MP login leaves up to a 60s window. In the normal termination flow the MP login is disabled, which drops the window to ~0s (tool calls fail at MP). Tripwire: operational pattern that revokes MCP-group access without touching the MP account, or a compliance ask for sub-minute propagation.

## Post-deploy IPv6 fix (PR #15)

First deploy of pass 2 surfaced `ERR_ERL_KEY_GEN_IPV6` at startup ‚Äî the unauthenticated fallback in the rate-limit `keyGenerator` was returning `req.ip` directly, so an IPv6 client could rotate through their /64 and bypass the 120 req/min cap. Server kept running (validation warning, not fatal), but the limit was real. One-line fix: route IP fallback through `ipKeyGenerator()` which normalizes IPv6 to /64 subnets. Redeployed clean.

## Pass 3 ‚Äî `claude/security-audit-pass-3` (fresh audit after deploy)

Another read of the codebase now that the pass-2 changes were in place. Looking specifically for regressions introduced by our own fixes and for spots I hadn't covered yet (CI, tsconfig, gitignore, edge cases in the new caching/rate-limit code).

### Real finding introduced in pass 2

- **N1 (MEDIUM-HIGH, fixed)** The `mcpRateLimit` keyGenerator bucketed every presented `Bearer xxx` value by its hash, verified or not. An unauthenticated attacker rotating random Bearer headers would get a fresh 120-req/min budget per rotation ‚Äî effectively unlimited. Now the hash bucket only kicks in when the token is in `verifyCache` (i.e., previously verified); everything else falls back to `ipKeyGenerator(req.ip)`. Legitimate users pay the IP bucket for their first request and then bucket by token hash ‚Äî strictly tighter, not looser, for the happy path.

### Edge-case polish

- **N3 (LOW, fixed)** `verifyCache` eviction was FIFO disguised as LRU ‚Äî V8 preserves the original Map slot on `set()` over an existing key, so a re-verified token stayed in its old position and could be evicted before newer-but-staler entries. Now delete-before-set so re-verified tokens move to the newest slot.
- **N4 (LOW, fixed)** `touchTransport` now runs before the transport lookup in `handleMcp`, closing a tiny race where the 30-min idle sweeper could decide to close a transport between our lookup and `handleRequest`.
- **N5 (LOW, fixed)** `userinfo.sub` is now quote-escaped before interpolating into the `dp_Users` filter. MP issues UUID subs so this was belt-and-suspenders, but free.

### CI hygiene

- **N6 (LOW, fixed)** `aquasecurity/trivy-action@master` pinned to the 0.35.0 commit SHA. A supply-chain change to the action can't land without review.
- **N7 (LOW, fixed)** CI `npm audit` now fails on `--audit-level=high`, matching the bar we've been holding in local checks (was `critical`).

### Investigated, no action

- **Rate-limit on `/health`** ‚Äî spammable, returns 30 bytes; monitoring endpoints are expected to be poll-friendly.
- **Graceful shutdown** ‚Äî in-flight requests drop on SIGTERM. Not a security concern.
- **Container binds 0.0.0.0** ‚Äî fine behind the Cloudflare tunnel; binding to the tunnel interface would be defense in depth, not closing a real gap.
- **tsconfig `strict: true`, `.gitignore`, CI permissions, `.trivyignore`, Dockerfile runtime user** ‚Äî re-checked, all clean.

## New behavior callers should know

- OAuth clients registering a non-`https://claude.ai/api/mcp/auth_callback` redirect URI must be added to `ALLOWED_REDIRECT_URIS`.
- When `ALLOWED_USER_GROUP_IDS` is set, transient MP outages now cause auth to deny rather than fall through.
- Date inputs must match `YYYY-MM-DD` exactly; looser formats rejected by Zod.
- Search inputs containing `%`, `_`, or `[` now match literally, not as wildcards.
- `/mcp` rate-limited to 120 req/min per token; subsequent requests within 60s reuse the cached `AuthInfo`.
- Per-user MCP sessions auto-expire after 30 min idle.

## Decisions

- **One PR per pass, not one per commit** ‚Äî #13 (pass 1, 6 commits), #14 (pass 2, 9 commits), #15 (IPv6 hotfix, 1 commit), #16 (pass 3, 4 commits). Keeps each review tractable.
- **Accepted risks written up in-repo, not just ticketed** ‚Äî `docs/security-posture.md` is the live record so future reviewers see what was considered and why not fixed.
- **Per-column allowlist deferred** ‚Äî `query_table` stays a power-user tool gated by MP role security rather than adding a per-column `readColumns` config. Revisit if sensitive tables are re-added or misuse is observed.
- **60s cache TTL over tighter revocation (N2)** ‚Äî operator confirmed the revocation scenario that would matter (removing from MCP group without disabling MP login) isn't an operational pattern here; disabling the MP account is the standard path and drops the window to ~0s. Documented rather than shortened.
- **`.claude/plans` plan doc kept** ‚Äî local-only source record; in-repo docs are authoritative.
