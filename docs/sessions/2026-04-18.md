# 2026-04-18 — Security Audit & Hardening

## Summary

Two-pass security audit of the full codebase. Seventeen findings total — six triaged and fixed in a first pass (PR #13, `claude/security-audit-sec-fixes`), then a comprehensive second pass covering everything else on `claude/security-audit-pass-2`. Fourteen findings fixed in code, three explicitly accepted as documented risks. `npm audit` now reports zero vulnerabilities.

## Pass 1 — PR #13 (`claude/security-audit-sec-fixes`)

Targeted fixes requested by the user.

- **HIGH** — `/register` was echoing `OIDC_CLIENT_SECRET` to unauthenticated registrants and accepting arbitrary `redirect_uris`. Registrants now get a fresh random secret; redirect URIs must be `https://` and on an allowlist (`https://claude.ai/api/mcp/auth_callback` + `ALLOWED_REDIRECT_URIS`); client store capped at 1000 with oldest-eviction.
- **CRITICAL** — OAuth proxy logged full request bodies (auth codes, client secrets, refresh tokens) and truncated token-response fields. `verifyAccessToken` logged a token prefix. All removed.
- **MEDIUM** — Request middleware buffered and logged MCP response bodies (PII from people/group queries). Reduced to method/path.
- **HIGH** — `ALLOWED_USER_GROUP_IDS` group check fell through on any failure (non-OK response, zero-row lookup, network error). Now fail-closed.
- **MEDIUM** — Date inputs now require `YYYY-MM-DD`. New `escapeLikeValue()` helper escapes `%`, `_`, `[` in addition to doubling quotes; routed every LIKE call site through it.
- **LOW** — Deleted unused `src/auth/oidc.ts` and `src/auth/session.ts` (never imported); removed `SESSION_SECRET` from config/env.example/README; dropped `version` from `/health`.

## Pass 2 — `claude/security-audit-pass-2`

Comprehensive audit across SQL, MCP, API, OAuth, PII, and dependencies. Seventeen findings; the six above plus eleven new ones.

### HIGH

- **#1 `describe_table` leaking PII** — was calling the target table with `$top: 1` and returning the actual row values as "sampleValue". Now returns only field name + inferred type.
- **#2 Prune default allowlist** — removed `dp_Users`, `Background_Checks`, `Form_Responses` from `table-access.example.json`. `dp_Users` was the big one (password-hash columns, reset tokens); the group-check path queries it directly outside the allowlist, so auth is unaffected. README documents what was removed and why.
- **#3 `query_table` hardening** — accepted option (c): document the trust model. `query_table`'s raw `$filter`/`$select`/`$orderby` pass-through is the ad-hoc escape hatch. MP's role-based security is the authoritative column/join gate once a table is allowlisted. Written up in `docs/security-posture.md`.

### MEDIUM

- **#4/#5 PII in logs** — MP error bodies echo user-supplied filter/select fragments (names, emails). Truncated to 200 chars in `transport.ts` before logging or re-raising. Per-request user log switched from `userName` to `userId` (opaque GUID).
- **#7 Transports map leak** — `handleMcp` only cleaned up on `transport.onclose`; abandoned connections stayed forever. Added per-user activity timestamps, 30-min idle sweeper (unref'd), 500-user cap with oldest-eviction.
- **#8/#9 Rate limit and userinfo cache** — SDK only rate-limits `/register` and `/token`. Added 120 req/min per-token limit on `/mcp` and `/`, keyed by `sha256(token)`. `verifyAccessToken` now caches `AuthInfo` for 60s per token hash (cap 10k, oldest-eviction, JWT exp honored).

### LOW — batched cleanup

- **#10** Dockerfile now runs as `node:node` (not root).
- **#11** Dead `originalClientStore` assignment removed.
- **#13** `get_record`'s `id` now `z.number().int().positive()`.
- **#14** `npm audit fix` cleared GHSA-458j-xx4x-4375 (transitive hono).
- **#15** Removed unused `openid-client` dep.
- **#17** Comment on `trust proxy = 1` explaining the single-hop (Cloudflare tunnel) assumption.

## Accepted risks

Documented in `docs/security-posture.md`:

- **#6 Prompt injection via MP freeform data** — structural mitigation already in place (`Form_Responses` excluded by default, donations out entirely). Claude's own instruction-following is the primary defense; a code-level strip/filter would be partial and fragile. Tripwire: a demonstrated exploit or pressure to re-add freeform tables by default.
- **#12 `getClient` returns configured `OIDC_CLIENT_SECRET`** — not exposed over any current HTTP path (SDK's `/register` only returns secrets to registrants; no RFC 7592 endpoint). Tripwire: adding any client-read endpoint or SDK upgrade that exposes this.
- **#16 Silent fallback to example `table-access.json` in container** — the example is itself conservative; correct place to enforce presence is deployment config, not app code. Tripwire: operator reporting an unexpected exposure.

## New behavior callers should know

- OAuth clients registering a non-`https://claude.ai/api/mcp/auth_callback` redirect URI must be added to `ALLOWED_REDIRECT_URIS`.
- When `ALLOWED_USER_GROUP_IDS` is set, transient MP outages now cause auth to deny rather than fall through.
- Date inputs must match `YYYY-MM-DD` exactly; looser formats rejected by Zod.
- Search inputs containing `%`, `_`, or `[` now match literally, not as wildcards.
- `/mcp` rate-limited to 120 req/min per token; subsequent requests within 60s reuse the cached `AuthInfo`.
- Per-user MCP sessions auto-expire after 30 min idle.

## Decisions

- **One PR per pass, not one per commit** — pass 1 is PR #13; pass 2 will be a follow-up PR off it. Keeps each review tractable.
- **Accepted risks written up in-repo, not just ticketed** — `docs/security-posture.md` is the live record so future reviewers see what was considered and why not fixed.
- **Per-column allowlist deferred** — `query_table` stays a power-user tool gated by MP role security rather than adding a per-column `readColumns` config. Revisit if sensitive tables are re-added or misuse is observed.
- **`.claude/plans` plan doc kept** — local-only source record; in-repo docs are authoritative.
